# SaaS Customer Churn Diagnostics  
**Google-Sponsored Case Competition | 80/20 Framework**

## Overview

This project analyzes a sudden spike in churn at a SaaS company using a structured 80/20 diagnostic framework. The objective was to identify the primary drivers of churn and prioritize high-impact business interventions.

This repository will be updated with detailed analysis, visualizations, and documentation after a full review of the project materials.

---

## Problem Statement

A SaaS business experienced an unexpected increase in customer churn. Leadership needed a data-driven root-cause analysis to:

- Identify key churn drivers  
- Quantify their impact  
- Prioritize actionable retention strategies  

---

## Analytical Approach

The analysis followed a hypothesis-driven diagnostic methodology:

- Exploratory data analysis (EDA)
- Driver segmentation
- Impact quantification
- Prioritization matrix using 80/20 principles
